# QLOS

QLOS is a Common Lisp project for constructing and executing quantum circuits.
The current release provides an in-process state-vector simulator, a compact
Lisp circuit DSL, direct state and gate APIs, tests, and a benchmark workload.

For a beginner-oriented introduction to the concepts represented by the
simulator, see
[Quantum Computing With QLOS](docs/README_QUANTUM.md). For the staged
long-term architecture and roadmap, see
[QLOS Project Direction](docs/README_QLOS.md).

## Requirements

- A Common Lisp implementation supported by ASDF; development currently uses
  SBCL.
- Quicklisp with the `rove` test framework when running the test suite.

## Load QLOS

Start SBCL from the repository root and load the local ASDF system:

```lisp
(require :asdf)
(asdf:load-asd (truename "qlos.asd"))
(asdf:load-system "qlos")
```

Call exported symbols with the `qlos:` package prefix, or import the package
for an interactive session:

```lisp
(use-package :qlos)
```

## Build And Run A Circuit

`qcircuit` is the shortest way to define a circuit. It infers the circuit's
qubit count from the greatest qubit index in its forms.

```lisp
(defparameter *circuit*
  (qlos:qcircuit
    (h 0)
    (cnot 0 1)
    (measure 0)
    (measure 1)))

(multiple-value-bind (state measurements)
    (qlos:run-circuit *circuit*)
  (format t "Measurements: ~S~%" measurements)
  (format t "Probabilities: ~S~%" (qlos:probabilities state)))
```

`run-circuit` returns two values:

- The final `quantum-state`.
- Measurement results in execution order as `(QUBIT . BIT)` pairs.

Supported circuit forms are currently:

```lisp
(x qubit)
(h qubit)
(cnot control target)
(measure qubit)
```

## Build A Circuit Procedurally

Use the builder API when operations are generated by normal Lisp code:

```lisp
(let ((circuit (qlos:make-circuit 3)))
  (dotimes (qubit 3)
    (qlos:circuit-h circuit qubit))
  (qlos:circuit-cnot circuit 0 1)
  (qlos:circuit-measure circuit 0)
  (qlos:run-circuit circuit))
```

The builder functions mutate and return the circuit, which also permits
threading calls through other construction helpers.

Inspect a circuit with:

```lisp
(qlos:circuit-qubit-count *circuit*)
(qlos:circuit-operations *circuit*)
```

`circuit-operations` returns a copy in execution order.

## Use The State API Directly

Create a state and apply gates without constructing a circuit:

```lisp
(let ((state (qlos:make-zero-state 2)))
  (qlos:h state 0)
  (qlos:cnot state 0 1)
  (values (qlos:amplitudes state)
          (qlos:probabilities state)
          (qlos:normalized-p state)))
```

Gate and measurement functions mutate their state argument. Use
`copy-quantum-state` when executions need independent copies:

```lisp
(let* ((initial (qlos:make-zero-state 1))
       (working (qlos:copy-quantum-state initial)))
  (qlos:x working 0)
  (values initial working))
```

An existing state can be supplied to `run-circuit` when its qubit count matches
the circuit:

```lisp
(qlos:run-circuit *circuit*
                  :state (qlos:make-zero-state 2))
```

## Run The Tests

From the repository root:

```sh
sbcl --non-interactive \
  --eval '(require :asdf)' \
  --eval '(asdf:load-asd (truename "qlos.asd"))' \
  --eval '(asdf:test-system "qlos")'
```

Tests are defined in `tests/simulator.lisp` and use Rove.

## Run The Benchmark

Load QLOS, then run:

```lisp
(qlos:benchmark-simulator :qubits 20 :layers 4)
```

Optional arguments include:

- `:measure t` to append measurement operations.
- `:maximum-state-vector-bytes` to change the default 1 GiB allocation guard.

Build the workload without executing it:

```lisp
(qlos:make-benchmark-circuit :qubits 20 :layers 4 :measure nil)
```

Estimate state-vector payload before selecting a workload size:

```lisp
(qlos:estimated-state-vector-bytes 20)
```

The estimate excludes Lisp array and runtime overhead.

## Project Layout

```text
src/package.lisp    Public package and exported API
src/state.lisp      State creation, inspection, copying, and measurement
src/gates.lisp      Gate execution kernels
src/circuit.lisp    Circuit representation, builders, DSL, and execution
src/benchmark.lisp  Benchmark circuit generation and reporting
tests/              Rove test system
docs/               Learning material and long-term project direction
qlos.asd            ASDF system definitions
```

## Current Scope

The current public gate set is `X`, `H`, and `CNOT`, with computational-basis
measurement. QLOS does not yet provide repeated-shot aggregation, noise models,
hardware execution, circuit serialization, or compiler passes.
